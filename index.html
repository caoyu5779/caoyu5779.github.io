国债

国债的本质

国债的类型

  1. 凭证式国债: 去银行买国债，银行给你凭证作为你借出钱的证明。这类国债往往不能在市场上流通，因为凭证仅仅被视为“收据”而不是“发票”。如果有需要，你可以拿凭证式国债作为银行的抵押贷款。
  2. 电子式国债: 储蓄国债，与凭证式国债类似。只是由实物凭证变为电子凭证。与凭证式国债相同，都可以提前支取，但是要收取0.1%的手续费，并按持有时间分档计息。
    1. 不满6个月提前兑取不计付利息
    2. 满6个月不满24个月按票面利率计息并扣除180天利息
    3. 满24个月不满36个月按票面利率计息并扣除90天利息
    4. 持有第二期满36个月不满60个月按票面利率计息并扣除60天利息

    EXP: 张大叔买了某期收益率为4.5%的凭证式国债，一共1000元，期限是3年，但是他急着用钱，刚到两年就要取出，这是他需要扣除的利息按照上面的“3”计算：

    1. 提前支取扣费公式=本金✖️利率/365天✖️扣息天数=1000✖️4.5%/365✖️90=11元
    2. 扣除的手续费=1000✖️0.1%=1元
    3. 一共需要扣除的费用是12元
    4. 当时获得的利息是1000✖️4.5%✖️2 = 90元
    5. 最终利息收益 90-12=78元，利息比例为 86.7%

提前支取扣除比例还是很高的，凭证式或电子式国债不建议提前支取

  1. 记账式国债: 通过电脑记账的方式来记录国家欠你的钱。1. 它可以上市流通，通过股票账户可以买到它。2. 未到期之前，它不能支取，但可以通过股票账户把它卖掉

EXP: 王大叔买了某期收益率为4%的记账式国债，一共1w元，期限是3年，刚到2年突然急着用钱，又不能提前支取怎么办？
他可以通过股票账户把它卖给XX投资人，获得变现，然后XX投资人再持有1年的时间可以到期获得利息

Linux编程

TCP/IP协议族是一个四层协议系统，自底而上分别是链路层、网络层、传输层和应用层。

数据链路层

实现了网卡接口的网络驱动程序，以处理数据在物理媒介上的传输。不同的物理网络具有不同的电气特性，网络驱动程序隐藏了这些细节，为上层协议提供一个统一的接口

链路层，常用协议分别是ARP协议(Address Resolve Protocol, 地址解析协议)和RARP协议(Reverse Address Resolve Protocol，逆地址解析协议)

网络层

ICMP协议(Internet Control Message Protocol, 因特网控制报文协议), 它是IP协议的重要补充，主要用于检测网络连接。ICMP协议使用的报文格式:8位类型+8位代码+16位检验+报文内容

报文类型分为两大类: 一类是差错报文，这类报文主要用来回应网络错误，比如目标不可到达和重定向。 另一类是查询报文，这类报文用来查询网络信息。

ARP协议原理: 主机向自己所在的网络广播一个ARP请求，该请求包含目标机器的网络地址。此网络上的其他机器都将收到这个请求，但只有被请求的目标机器会回答一个ARP应答，其中包含自己的物理地址
